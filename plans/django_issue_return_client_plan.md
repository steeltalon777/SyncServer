# План: гайд по выдаче и возврату выдачи для Django-клиента

## Цель

Подготовить практический markdown-гайд для Django-клиента, который описывает HTTP-интеграцию с текущим API SyncServer для сценариев:

- выдача актива сотруднику через операцию `ISSUE`
- возврат ранее выданного актива через операцию `ISSUE_RETURN`

## Подтверждённые источники

- `POST /api/v1/operations`, `POST /api/v1/operations/{operation_id}/submit`, `POST /api/v1/operations/{operation_id}/accept-lines` в актуальном роутере операций
- `GET /api/v1/issued-assets` для проверки текущего объёма выданного актива у получателя
- `GET /api/v1/recipients`, `POST /api/v1/recipients` для выбора или создания получателя
- Интеграционные тесты, подтверждающие рабочий сценарий `ISSUE` и `ISSUE_RETURN`

## Подтверждённые факты по текущему API

- Операции выдачи и возврата используют один и тот же общий поток: создать черновик операции, затем выполнить submit
- Для `ISSUE` сервер при submit уменьшает остаток на складе и увеличивает запись в реестре выданных активов
- Для `ISSUE_RETURN` сервер при submit уменьшает запись в реестре выданных активов и возвращает количество на склад
- Для `ISSUE` и `ISSUE_RETURN` не используется шаг приёмки строк через `accept-lines`; этот endpoint нужен для `RECEIVE` и `MOVE`
- Получатель может быть передан через `recipient_id` либо через имя; при передаче имени сервер способен создать или переиспользовать получателя
- Для проверки результата возврата клиенту достаточно повторно запросить реестр выданных активов по `recipient_id`
- На создание draft достаточно прав operate на сайт, но submit разрешён только `chief_storekeeper` и `root`
- В части сводной документации есть устаревшее расхождение: старое описание упоминает `501` для `ISSUE` и `ISSUE_RETURN`, но актуальные роуты, сервисы и тесты подтверждают рабочий сценарий

## Границы будущего гайда

- без полноценной реализации `views`, `forms`, `templates`

## Примечание

Гайд должен опираться на текущее поведение кода и тестов, а не только на существующие сводные документы, так как часть старых ссылок по `ISSUE` и `ISSUE_RETURN` выглядит устаревшей.
