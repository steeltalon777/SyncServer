# ADR-0004: Soft deactivation for catalog entities via is_active

## Status
Accepted

## Context
Каталог используется клиентами синхронизации; физическое удаление справочников может ломать ссылки в уже синхронизированных данных и исторических событиях.

## Decision
Для `Category`, `Unit`, `Item` применять soft delete-подход: изменение `is_active=false` вместо физического удаления строки.

## Consequences
Плюсы:
- Сохраняется referential integrity для старых записей и событий.
- Проще аудит и восстановление деактивированных записей.
- Безопаснее для офлайн-клиентов с отложенной синхронизацией.

Минусы:
- Нужна дисциплина фильтрации `is_active` в сценариях чтения.
- Таблицы растут без физической очистки.

## Alternatives Considered
### Option 1: Hard delete
Освобождает таблицы, но создаёт риск потери связности и истории.

### Option 2: Archive tables
Гибко, но увеличивает сложность data flow и сопровождения.
